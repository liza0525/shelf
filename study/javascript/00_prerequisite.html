JavaScript 시작 전 환경 설정

VSCode 기준, Windows / Linux / macOS 차이를 항목별로 정리합니다. ← → 키로 페이지를 넘길 수 있어요.

들어가며

Node 버전 관리자가 OS마다 다른 도구다

JavaScript는 브라우저/Node 어디서나 실행 파일 자체는 동일하지만, "여러 Node 버전을 설치하고 전환하는 도구"가 Windows와 Linux/macOS에서 서로 다른 프로그램이에요. 이 차이를 모르고 튜토리얼을 그대로 따라 하면 명령어가 안 먹히는 경우가 많습니다.

주의

  • nvm(Linux/macOS)과 nvm-windows는 다른 프로젝트
  • PowerShell 실행 정책이 npm 스크립트를 막을 수 있음
  • Windows 긴 경로 문제(node_modules 중첩)

이 자료의 기준

  • Node.js LTS 버전 사용
  • VSCode + ESLint + Prettier 확장 기준
  • OS별 버전 관리자를 각각 안내

준비물

설치할 4가지

01
VSCode 에디터 겸 통합 터미널, 디버거 환경.
02
ESLint / Prettier 확장 문법 검사와 코드 포맷을 자동화.
03
Node.js (LTS) + npm 런타임과 패키지 매니저. npm은 Node에 기본 포함.
04
버전 관리자 (nvm 계열) OS별로 다른 도구를 씁니다.

VSCode 설치

OS별 설치 방법

Windows

  • 공식 installer(.exe) 실행
  • "PATH에 추가" 체크박스 체크
winget install Microsoft.VisualStudioCode

Linux

  • .deb/.rpm 또는 apt 저장소 등록
  • code 명령어 자동 등록됨
sudo apt install code # 또는 snap install code --classic

macOS

  • .dmg를 Applications로 드래그
  • code 명령은 팔레트에서 별도 등록
brew install --cask visual-studio-code # 이후 Cmd+Shift+P # "Shell Command: Install 'code'"

Node.js 설치

Node.js LTS 버전 설치

Windows

  • winget 또는 공식 .msi installer
  • npm은 자동으로 함께 설치됨
winget install OpenJS.NodeJS.LTS

Linux

  • 배포판 기본 저장소는 버전이 낮을 수 있음
  • NodeSource 저장소 등록 권장
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash - sudo apt install nodejs

macOS

  • brew로 간단 설치
  • 버전 관리 원하면 nvm 권장
brew install node

버전 관리자

nvm vs nvm-windows, 다른 프로그램입니다

이름은 비슷하지만 Windows용은 커뮤니티가 별도로 만든 완전히 다른 도구예요. 명령어는 비슷해도 설치 방식과 동작이 다릅니다.

Windows (nvm-windows)

  • coreybutler/nvm-windows, 관리자 권한 설치 필요
  • .nvmrc 자동 인식 안 됨
winget install CoreyButler.NVMforWindows nvm install lts nvm use 20.x.x

Linux (nvm)

  • 공식 nvm.sh 스크립트
  • .nvmrc 자동 인식(nvm use)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash nvm install --lts nvm use --lts

macOS (nvm)

  • Linux와 완전히 동일한 스크립트
  • brew install nvm으로도 설치 가능
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash nvm install --lts

환경 설정 차이

Windows만 겪는 두 가지 함정

PowerShell 실행 정책

npm.ps1 실행이 차단될 수 있음

"이 시스템에서 스크립트를 실행할 수 없습니다" 오류가 뜨면 PowerShell(관리자)에서 Set-ExecutionPolicy RemoteSigned -Scope CurrentUser 실행. Linux/macOS의 bash/zsh는 이런 제약이 없음.

경로 길이 제한

중첩된 node_modules에서 에러 가능

Windows 10 이상은 레지스트리에서 LongPathsEnabled를 켜면 대부분 해결. Linux/macOS는 경로 길이 제한이 훨씬 넉넉해 신경 쓸 필요 없음.

설치 확인

버전 체크로 마무리

터미널 (모든 OS 공통 명령)

node -v npm -v # nvm 사용 중이면 which node # macOS/Linux where.exe node # Windows

node -v가 nvm으로 설치한 버전이 아니라면, 시스템에 기본 설치된 Node가 먼저 잡히고 있는 경우가 많습니다.

자주 걸리는 문제

OS별 트러블슈팅

Win
npm run dev가 "실행할 수 없음"으로 막힘 PowerShell 실행 정책 문제. RemoteSigned로 바꾸거나 VSCode 터미널을 cmd로 바꿔서 실행.
Linux
sudo npm install -g 시 권한 꼬임 전역 설치를 sudo로 하면 이후 권한 문제가 반복됨. nvm으로 설치한 Node는 sudo 없이 전역 설치 가능.
macOS
brew node와 nvm node가 충돌 두 가지 방법을 동시에 쓰면 which node 결과가 매번 달라짐. 하나만 선택해서 사용하는 것을 권장.
공통
package-lock.json 충돌 npm 버전이 서로 다르면 lock 파일 포맷이 미묘하게 달라짐. 팀 전체가 같은 Node LTS 버전을 쓰는 게 안전.

VSCode 통합 터미널

기본 셸, 어떤 걸 쓸까

Windows

PowerShell 또는 Git Bash

기본은 PowerShell. Linux 셸 문법(&& 뒤 조건 등)에 익숙하다면 Git Bash를 기본 프로필로 지정해도 좋음.

Linux / macOS

bash 또는 zsh 그대로

별도 설정 없이 시스템 기본 셸이 열림. macOS 기본은 zsh.

실습

VSCode에서 첫 Node 스크립트 실행하기

01
빈 폴더 생성 후 code . 로 VSCode 열기 mkdir hello-js && cd hello-js && code .
02
npm init -y 로 package.json 생성 기본값으로 초기화 후 index.js 파일 생성.
03
console.log("Hello, JS!") 작성 후 실행 통합 터미널에서 node index.js.
04
F5로 디버거 실행해보기 "Run and Debug" → Node.js 자동 감지. 중단점을 걸고 값이 보이면 연동 성공.

마무리

1주차 들어가기 전 셀프 체크

node -v, npm -v가 오류 없이 출력되나요?

nvm으로 설치했다면 which node 결과도 함께 확인하세요.

내 OS의 버전 관리자 이름이 nvm인지 nvm-windows인지 알고 있나요?

두 도구는 이름만 비슷할 뿐 설치·명령어 방식이 다릅니다.

Windows라면 PowerShell 실행 정책 문제를 겪지 않았나요?

npm 스크립트가 막히면 RemoteSigned 정책으로 바꿔보세요.

ESLint/Prettier가 저장할 때 자동으로 동작하나요?

settings.json에 "editor.formatOnSave": true를 켜두면 편해요.
