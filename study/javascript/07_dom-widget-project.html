DOM BASICS

DOM, 선택하고 조작하기

브라우저는 HTML을 트리 구조(DOM)로 표현합니다. JS로 이 트리에서 노드를 찾아 클래스와 텍스트를 바꾸는 것이 화면 조작의 전부입니다.

// CSS 선택자로 요소 하나 / 여러 개 찾기 const title = document.querySelector("h1"); const items = document.querySelectorAll(".item"); // 클래스 추가·제거·토글 — 조건부 스타일링에 사용 title.classList.add("active"); title.classList.remove("hidden"); title.classList.toggle("done"); // 텍스트/속성 변경 title.textContent = "완료됨"; title.setAttribute("data-status", "done");

querySelector

일치하는 첫 요소 하나

Python으로 치면 리스트에서 첫 항목만 꺼내는 느낌 — 없으면 null 반환

querySelectorAll

일치하는 모든 요소

NodeList 반환 — forEach는 가능하지만 배열 메서드 전부는 아님, 필요하면 Array.from()

EVENTS

클릭에 반응하기

addEventListener는 "이 요소에서 이런 이벤트가 발생하면 이 함수를 실행해줘"라는 등록입니다. 콜백은 event 객체를 인자로 받습니다.

const btn = document.querySelector("#save-btn"); btn.addEventListener("click", (event) => { console.log("클릭됨:", event.target); }); // 리스너를 나중에 떼려면 콜백을 변수로 빼둬야 한다 function handleClick(event) { /* ... */ } btn.addEventListener("click", handleClick); btn.removeEventListener("click", handleClick);

자주 쓰는 이벤트: click, submit, input, change, keydown

EVENT DELEGATION

자식마다 말고, 부모 하나로

이벤트는 자식에서 부모로 버블링됩니다. 목록 항목 100개에 리스너 100개를 달 필요 없이, 부모 하나에만 걸고 event.target으로 실제 클릭된 자식을 판별합니다.

전: 자식마다 리스너 등록 (문제 있음)

document.querySelectorAll(".delete-btn").forEach((btn) => { btn.addEventListener("click", (e) => { e.target.closest(".item").remove(); }); }); // 문제: 나중에 추가된 항목의 버튼에는 리스너가 안 붙는다

후: 부모 하나에 위임 (버블링 활용)

list.addEventListener("click", (event) => { if (!event.target.matches(".delete-btn")) return; event.target.closest(".item").remove(); }); // 새 항목이 추가돼도 항상 동작 — 리스너는 하나뿐

FORMS

submit 이벤트와 preventDefault

<form>은 기본적으로 제출 시 페이지를 새로고침합니다. 서버 왕복 없이 JS가 값을 직접 처리하려면 이 기본 동작을 막아야 합니다.

const form = document.querySelector("#todo-form"); const input = document.querySelector("#todo-input"); form.addEventListener("submit", (event) => { event.preventDefault(); // 새로고침 방지 const text = input.value.trim(); if (text === "") return; addTodo(text); input.value = ""; // 입력창 비우기 });

Python Flask/Django의 폼 제출과 달리, 서버로 보내지 않고 브라우저 안에서 끝냅니다.

STATE → RENDER

프레임워크가 하는 일, 손으로 해보기

React/Vue는 상태가 바뀌면 자동으로 화면을 다시 그립니다. 바닐라 JS에서는 이 흐름을 직접 만들어야 합니다.

상태 → 렌더 함수 → DOM, 항상 이 방향으로만 흐름

DATA MODEL

배열 of 객체로 상태 표현하기

화면에 보이는 목록은 이 배열을 그대로 그린 결과일 뿐입니다. "진실의 원천(source of truth)"은 DOM이 아니라 이 배열입니다.

let todos = [ { id: 1, text: "장보기", done: false }, { id: 2, text: "빨래하기", done: true }, ]; // 새 id는 현재 시각(ms)으로 충돌 없이 생성 function createTodo(text) { return { id: Date.now(), text, done: false }; }

Python으로 치면 dict를 담은 list와 동일한 구조입니다.

CRUD FUNCTIONS

추가 · 삭제 · 완료 토글

세 함수 모두 배열(state)만 바꾸고, 마지막에 render()를 호출해 화면을 상태와 동기화합니다.

function addTodo(text) { todos.push(createTodo(text)); render(); } function removeTodo(id) { todos = todos.filter((todo) => todo.id !== id); render(); } function toggleTodo(id) { todos = todos.map((todo) => todo.id === id ? { ...todo, done: !todo.done } : todo ); render(); }

PERSISTENCE

localStorage로 새로고침해도 안 사라지게

localStorage는 문자열만 저장할 수 있습니다. 객체/배열은 JSON.stringify로 직렬화하고, 꺼낼 때 JSON.parse로 복원합니다.

const STORAGE_KEY = "todos"; function saveTodos() { localStorage.setItem(STORAGE_KEY, JSON.stringify(todos)); } function loadTodos() { const raw = localStorage.getItem(STORAGE_KEY); return raw ? JSON.parse(raw) : []; } let todos = loadTodos(); function render() { // ...목록 DOM을 다시 그리는 코드... saveTodos(); // 상태가 바뀔 때마다 함께 저장 }

PITFALLS

자주 하는 실수

01

리스너 중복 등록 render()가 호출될 때마다 addEventListener를 다시 걸면 클릭 한 번에 핸들러가 여러 번 실행됩니다. 목록 항목에는 이벤트 위임을 써서 부모에 한 번만 등록하세요.

02

JSON.stringify 없이 저장 localStorage.setItem은 문자열만 받습니다. 객체를 그대로 넣으면 "[object Object]"라는 문자열로 저장되어 다시 불러올 수 없습니다.

03

state만 바꾸고 render() 호출을 빠뜨림 todos.push()만 하고 render()를 안 부르면 데이터는 바뀌었는데 화면은 그대로입니다. 상태를 바꾸는 함수 끝에는 항상 render()를 잊지 마세요.

04

사용자 입력을 innerHTML에 그대로 삽입 input.value를 이스케이프 없이 innerHTML에 넣으면 XSS 위험이 있습니다. 텍스트만 넣을 때는 textContent를 사용하세요.
