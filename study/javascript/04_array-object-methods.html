배열 메서드 · map

map — 각 원소를 변환

// 배열의 모든 원소를 변환해 새 배열을 반환한다 const prices = [1000, 2000, 3000]; const withTax = prices.map((price) => price * 1.1); // [1100.000000000001, 2200, 3300] // 인덱스도 두 번째 인자로 받을 수 있다 const tagged = prices.map((price, i) => `#${i}: ${price}`);

Python 대비

list comprehension vs map

Python

  • [p * 1.1 for p in prices]
  • list(map(lambda p: p*1.1, prices))
  • 컴프리헨션이 더 관용적인 표현

JavaScript

  • prices.map(p => p * 1.1)
  • 컴프리헨션 문법 자체가 없다
  • map/filter/reduce가 그 자리를 대신한다

배열 메서드 · filter / reduce

filter로 거르고 reduce로 모은다

// filter: 조건을 만족하는 원소만 남긴다 (Python의 [x for x in xs if ...]) const nums = [1, 2, 3, 4, 5, 6]; const evens = nums.filter((n) => n % 2 === 0); // [2, 4, 6] // reduce: 누적값을 만든다 (Python의 functools.reduce와 동일한 개념) const total = nums.reduce((acc, n) => acc + n, 0); // 초기값 0 필수! // acc: 누적값, n: 현재 원소 → total === 21

체이닝

세 메서드를 이어붙이기

// filter → map → reduce로 파이프라인처럼 연결한다 const orders = [ { item: "keyboard", price: 50000, qty: 2 }, { item: "mouse", price: 20000, qty: 1 }, { item: "monitor", price: 300000, qty: 1 }, ]; const total = orders .filter((o) => o.price >= 30000) .map((o) => o.price * o.qty) .reduce((sum, n) => sum + n, 0); // keyboard(100000) + monitor(300000) = 400000

구조 분해 할당 · 배열

배열 구조 분해

// 순서대로 변수에 대입된다 (Python의 튜플 언패킹과 유사) const [first, second] = ["apple", "banana", "cherry"]; // first === "apple", second === "banana" // 건너뛰기, 기본값, rest도 가능하다 const [a, , c] = [1, 2, 3]; // a: 1, c: 3 const [x = 10] = []; // x: 10 (기본값) const [head, ...tail] = [1, 2, 3]; // head: 1, tail: [2, 3]

구조 분해 할당 · 객체

객체 구조 분해

// key 이름으로 꺼낸다 (순서 무관, Python에 정확한 대응 문법 없음) const user = { name: "지영", age: 28, city: "서울" }; const { name, age } = user; // name: "지영", age: 28 // 이름 바꾸기(renaming) + 기본값 const { name: userName, country = "KR" } = user; // userName: "지영", country: "KR" (원본에 없어서 기본값 사용) // 함수 매개변수에서도 바로 구조 분해할 수 있다 function greet({ name, age }) { return `${name}(${age}세)님 안녕하세요`; }

spread 문법

...으로 펼치고 합치기

// 배열 합치기 (Python의 [*a, *b] 와 동일한 발상) const a = [1, 2]; const b = [3, 4]; const merged = [...a, ...b]; // [1, 2, 3, 4] // 객체 합치기 (Python의 {**a, **b}) const defaults = { theme: "light", lang: "ko" }; const settings = { ...defaults, theme: "dark" }; // { theme: "dark", lang: "ko" } — 뒤에 오는 값이 덮어쓴다 // 함수 호출에도 펼쳐서 인자로 넘길 수 있다 Math.max(...[3, 1, 4, 1, 5]); // 5

함정 주의

얕은 복사 vs 깊은 복사

// spread는 "한 겹"만 복사한다 — 얕은 복사(shallow copy) const original = { name: "지영", address: { city: "서울" } }; const copy = { ...original }; copy.name = "민수"; // 원본에 영향 없음 (최상위는 안전) copy.address.city = "부산"; // !! original.address.city도 "부산"으로 바뀐다 // 중첩 객체는 참조를 그대로 공유하기 때문 // 깊은 복사가 필요하면: const deepCopy = structuredClone(original); // 최신 런타임 내장 함수 // 또는 JSON.parse(JSON.stringify(original)) (함수·undefined는 유실됨)

?. 와 ??

옵셔널 체이닝 & nullish coalescing

// ?. : 중간 값이 null/undefined면 에러 대신 undefined 반환 const user = { profile: { nickname: "치즈" } }; const nick = user.profile?.nickname; // "치즈" const age = user.profile?.age?.toFixed(); // undefined (에러 없음) // ?? : 왼쪽이 null/undefined일 때만 오른쪽 값을 쓴다 (|| 와 다름! 0, ""도 살린다) const count = user.profile?.count ?? 0; // undefined -> 0 const label = "" ?? "기본값"; // "" (falsy지만 null/undefined 아님)

Python 비교

Python

  • data.get("profile", {}).get("nickname")
  • dict.get(key, default) 로 안전 접근

JavaScript

  • data.profile?.nickname ?? "기본값"
  • 문법 자체로 안전 접근 + 기본값 지정

실습 미리보기

JSON 데이터 가공하기

// 구조 분해 + 옵셔널 체이닝 + 메서드 체이닝을 한 번에 function summarize(users) { return users .filter(({ active }) => active) .map(({ name, address }) => ({ name, city: address?.city ?? "미상", })); } const users = [ { name: "지영", active: true, address: { city: "서울" } }, { name: "민수", active: false, address: { city: "대전" } }, { name: "하늘", active: true }, ]; summarize(users); // [{ name: "지영", city: "서울" }, { name: "하늘", city: "미상" }]

흔한 실수

이건 꼭 조심하세요

01

reduce 초기값을 빠뜨리기 초기값 없이 reduce를 쓰면 빈 배열에서 TypeError가 나거나, 첫 원소가 그대로 acc가 되어 의도치 않은 타입이 섞입니다. 항상 두 번째 인자로 초기값을 넘기세요.

02

spread를 깊은 복사로 착각하기 { ...obj }는 한 겹만 복사합니다. 중첩된 객체·배열을 수정하면 원본도 함께 바뀌는 버그로 이어지기 쉽습니다.

03

?? 대신 ||를 습관적으로 쓰기 ||는 0, "", false도 걸러내 버립니다. "값이 아예 없을 때만" 기본값을 쓰고 싶다면 ??가 정답입니다.
