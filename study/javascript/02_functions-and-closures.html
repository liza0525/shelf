함수 정의 · 세 가지 문법

함수를 만드는 세 가지 방법

Python엔 def 하나뿐이지만, JS는 상황에 따라 세 가지 문법을 골라 쓴다.

함수 선언문

function 키워드

function add(a, b) { return a + b; } — 코드 어디서든 먼저 정의된 것처럼 호출할 수 있다(hoisting). Python def와 가장 비슷한 형태.

함수 표현식

변수에 담긴 함수

const add = function(a, b) { return a + b; }; — 변수에 할당되므로, 정의보다 먼저 호출하면 에러가 난다.

화살표 함수

() => {}

const add = (a, b) => a + b; — ES6에 추가된 축약 문법. 중괄호 없이 쓰면 return이 생략된다(암묵적 반환).

화살표 함수 · Arrow Function

화살표 함수는 단순한 축약형이 아니다

문법만 짧은 게 아니라, 일반 함수와는 동작 자체가 다른 지점이 세 가지 있다.

01

자신만의 this가 없다

화살표 함수 안의 this는 정의된 시점의 바깥 스코프 this를 그대로 이어받는다(렉시컬 this). 콜백 안에서 특히 유용하다.

02

arguments가 없다

일반 함수와 달리 화살표 함수 내부에는 arguments 객체가 존재하지 않는다. 필요하면 rest 파라미터(...args)를 쓴다.

03

생성자로 쓸 수 없다

화살표 함수는 new로 호출할 수 없다. 항상 값(콜백, 짧은 유틸 함수)으로만 쓰인다.

this가 호출 방식에 따라 달라지는 규칙은 3주차(객체와 프로토타입, this)에서 자세히 다룬다 — 오늘은 "화살표 함수는 this를 새로 만들지 않는다"는 것만 기억하자.

일급 객체 · First-class Function

함수를 값처럼 다루는 건 Python도 마찬가지다

다만 실제 코드에서는 함수를 콜백으로 넘기는 패턴이 JS 쪽에서 훨씬 더 일상적으로 등장한다.

Python

  • 함수도 객체라 변수·인자로 자유롭게 전달
  • map / filter / sorted(key=...) 정도가 대표적 사례
  • 콜백보다는 반환값을 그대로 쓰는 동기 스타일이 흔함
  • 데코레이터(@)도 결국 함수를 감싸는 함수

JavaScript

  • 함수도 객체라 변수·배열·객체 속성에 자유롭게 저장
  • addEventListener, setTimeout, 배열 메서드가 기본 콜백 패턴
  • 과거의 비동기 처리 자체가 콜백을 넘기는 방식으로 동작
  • 함수를 인자/반환값으로 주고받는 일이 훨씬 잦음

콜백 패턴 · Callback

함수를 인자로 넘기는 것이 일상이다

타이머 콜백 — setTimeout

function greet(name) { console.log(`안녕, ${name}!`); } setTimeout(function () { greet("world"); }, 1000); // 1초 뒤 실행할 함수를 통째로 넘긴다

배열 메서드 콜백 — forEach

const nums = [1, 2, 3]; nums.forEach((n) => console.log(n * 2)); // 화살표 함수를 콜백으로

클로저 · Closure

함수는 자신이 태어난 스코프를 기억한다

클로저(closure)란 함수와, 그 함수가 정의될 당시의 렉시컬 스코프(lexical scope)를 함께 묶은 것이다.

STEP 1

outer() 호출

outer 함수가 실행되며 count 변수가 생성된다

STEP 2

inner 함수 생성

outer 내부에서 정의된 inner가 count를 캡처한다

STEP 3

outer() 종료

보통이면 count는 사라져야 하지만, inner가 참조 중이라 살아남는다

STEP 4

inner() 재호출

outer가 이미 끝났어도 inner는 count에 계속 접근/수정한다

클로저 · 카운터 함수

카운터 함수로 보는 클로저

makeCounter — 호출할 때마다 증가하는 카운터

function makeCounter() { let count = 0; // 이 변수는 makeCounter 스코프에 갇힌다 return function () { count += 1; // 클로저가 count를 계속 기억하고 참조한다 return count; }; } const counter = makeCounter(); console.log(counter()); // 1 console.log(counter()); // 2 console.log(counter()); // 3 const counter2 = makeCounter(); console.log(counter2()); // 1 — counter와 count를 공유하지 않는다

클로저 · Python 대비

Python도 클로저가 있다, 문법이 다를 뿐

두 언어 모두 렉시컬 스코프를 기반으로 클로저를 지원하지만, 바깥 변수를 "재대입"할 때 요구하는 문법이 다르다.

Python

  • 바깥 변수를 재대입하려면 nonlocal 선언 필요
  • 선언을 빠뜨리면 새 지역 변수로 취급되어 오류 발생
  • 함수도 일급 객체라 클로저 자체는 자연스럽게 지원
  • def make_counter(): 안에서 nonlocal count 필요

JavaScript

  • let/const로 선언한 변수는 별도 키워드 없이 재대입 가능
  • 클로저는 변수의 값이 아니라 변수 자체(참조)를 캡처
  • function, 화살표 함수 모두 자유롭게 클로저를 만듦
  • var를 쓰면 함수 스코프라 예상과 다르게 동작할 수 있음

매개변수 · Default & Rest

기본값과 나머지 매개변수

Python의 기본값 인자, *args와 결이 비슷하지만 세부 동작은 다르다.

기본 매개변수 (default parameter)

function greet(name = "익명") { return `안녕, ${name}!`; } console.log(greet()); // "안녕, 익명!" console.log(greet("아영")); // "안녕, 아영!"

arguments

유사 배열 객체 (구식)

function으로 선언한 함수 안에서만 쓸 수 있는 특수 객체. 진짜 배열이 아니라서 배열 메서드를 바로 못 쓴다. 화살표 함수에는 존재하지 않는다.

...rest

진짜 배열 (ES6+)

function sum(...nums) {}처럼 나머지 인자를 진짜 Array로 모아준다. 화살표 함수에서도 쓸 수 있고, Python의 *args와 가장 비슷하다.

실습 미리보기 · Memoization

클로저로 캐시 함수 만들기

느린 계산 결과를 클로저 안에 저장해두면, 같은 입력에 대해 다시 계산하지 않고 캐시된 값을 즉시 돌려줄 수 있다.

function memoize(fn) { const cache = new Map(); // 클로저에 갇히는 캐시 저장소 return function (...args) { const key = JSON.stringify(args); if (cache.has(key)) { console.log("캐시 사용:", key); return cache.get(key); } const result = fn(...args); cache.set(key, result); return result; }; } function slowSquare(n) { for (let i = 0; i < 1e8; i++) {} // 일부러 느리게 return n * n; } const fastSquare = memoize(slowSquare); fastSquare(5); // 느리게 계산한 뒤 캐시에 저장 fastSquare(5); // 캐시에서 즉시 반환

실습 과제: 위 memoize를 직접 구현하고, 카운터 함수와 함께 클로저 활용법을 익혀보세요.

클로저 · 흔한 실수

흔한 실수

01

var로 반복문 안에서 클로저 만들기 var는 함수 스코프라 반복문이 다 끝난 뒤의 값(예: 3)을 모든 클로저가 공유해서 참조한다. for (var i = 0; i < 3; i++) { setTimeout(() => console.log(i), 0); }는 0, 1, 2가 아니라 3, 3, 3을 출력한다.

02

해결책: let로 바꾸기 let은 반복마다 새로운 바인딩을 만들어서, 각 클로저가 자신만의 i를 캡처한다. var를 let으로만 바꿔도 의도대로 0, 1, 2가 출력된다.

03

"캡처한 순간의 값"이라고 착각하기 클로저는 변수의 스냅샷이 아니라 변수 자체(참조)를 기억한다. 클로저가 만들어진 뒤 바깥에서 변수를 바꾸면, 클로저 안에서 보는 값도 함께 바뀐다.
