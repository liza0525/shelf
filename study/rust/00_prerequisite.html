Rust 시작 전 환경 설정

VSCode 기준, Windows / Linux / macOS 차이를 항목별로 정리합니다. ← → 키로 페이지를 넘길 수 있어요.

들어가며

rustup은 같아도, 링커는 OS마다 다르다

rustup 설치 자체는 세 OS 모두 거의 동일해요. 하지만 cargo build가 내부적으로 호출하는 "링커"는 OS마다 별도로 준비해야 하는 도구라서, 이걸 빼먹으면 rustc는 성공해도 링크 단계에서 실패합니다.

링커가 없으면 나는 에러

  • error: linker `link.exe`/`cc` not found
  • Windows: MSVC Build Tools 미설치
  • macOS: Xcode Command Line Tools 미설치

이 자료의 기준

  • rustup + stable 툴체인
  • VSCode + rust-analyzer 확장
  • OS별 링커 준비 방법을 각각 안내

준비물

설치할 4가지

01
VSCode 에디터 겸 cargo 명령 실행, 디버거 환경.
02
rust-analyzer / CodeLLDB 확장 언어 서버와 디버거 연동.
03
rustup (rustc + cargo) 공식 툴체인 관리자, 세 OS 모두 거의 동일한 설치 과정.
04
OS별 링커 / 빌드 도구 Windows는 MSVC Build Tools, macOS는 Xcode CLT 필요.

VSCode 설치

OS별 설치 방법

Windows

  • 공식 installer(.exe) 실행
  • "PATH에 추가" 체크박스 체크
winget install Microsoft.VisualStudioCode

Linux

  • .deb/.rpm 또는 apt 저장소 등록
  • code 명령어 자동 등록됨
sudo apt install code # 또는 snap install code --classic

macOS

  • .dmg를 Applications로 드래그
  • code 명령은 팔레트에서 별도 등록
brew install --cask visual-studio-code # 이후 Cmd+Shift+P # "Shell Command: Install 'code'"

OS 상관없이 동일하게 설치합니다.

rustup 설치

rustc + cargo, 설치 명령은 OS별로 조금씩

Windows

  • rustup-init.exe 실행 (rustup.rs에서 다운로드)
  • winget으로도 설치 가능
winget install Rustlang.Rustup

Linux

  • 공식 셸 스크립트
  • 설치 후 터미널 재시작 필요
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

macOS

  • Linux와 동일한 스크립트
  • brew install rustup-init 으로도 가능
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

링커 준비 (중요)

cargo build가 실제로 호출하는 컴파일러

rustup만 설치하고 이 단계를 건너뛰면 "linker not found" 에러가 납니다. OS별로 준비물이 다릅니다.

Windows (MSVC, 기본값)

  • Visual Studio Build Tools 필요
  • "Desktop development with C++" 워크로드 체크
winget install Microsoft.VisualStudio.2022.BuildTools # 설치 시 C++ 빌드 도구 워크로드 선택

Linux

  • gcc가 링커 역할
  • build-essential에 포함
sudo apt install build-essential pkg-config libssl-dev

macOS

  • clang이 링커 역할
  • Xcode CLT 하나로 해결
xcode-select --install

호스트 triple

rustup show로 보이는 OS별 기본 타깃

8주차 웹 서버 프로젝트를 배포할 때 이 triple 차이를 알아야 크로스 컴파일 여부를 판단할 수 있어요.

Windows

x86_64-pc-windows-msvc

GNU 툴체인을 쓰면 -gnu 접미사. 기본은 -msvc.

Linux

x86_64-unknown-linux-gnu

대부분의 배포판에서 동일. musl 정적 빌드는 별도 타깃 추가 설치.

macOS

aarch64/x86_64-apple-darwin

Apple Silicon은 aarch64, Intel Mac은 x86_64로 갈림.

설치 확인

버전 체크 + 링커 동작 확인

터미널 (모든 OS 공통 명령)

rustc --version cargo --version rustup show # 새 프로젝트로 링커까지 실제로 테스트 cargo new hello_rust cd hello_rust cargo run

cargo run까지 "Hello, world!"가 출력되면 링커까지 전부 정상 동작하는 것입니다.

자주 걸리는 문제

OS별 트러블슈팅

Win
error: linker `link.exe` not found Visual Studio Build Tools 미설치. 설치 시 "Desktop development with C++" 워크로드를 반드시 선택해야 함.
Win(대안)
Visual Studio 설치가 부담스러운 경우 rustup default stable-x86_64-pc-windows-gnu로 전환 후 MSYS2의 mingw-w64 gcc를 링커로 사용하는 방법도 있음.
Linux
error: could not find system library 'openssl' 일부 크레이트가 OpenSSL 헤더를 요구. libssl-dev(Debian) 또는 openssl-devel(Fedora) 설치.
macOS
xcrun: error: invalid active developer path Xcode CLT가 없거나 손상됨. xcode-select --install을 다시 실행하면 해결.

rust-analyzer 확인

에디터에서 링크가 잘 잡혔는지 확인하기

VSCode 하단 상태 바에 "rust-analyzer" 아이콘이 뜨고, 코드에 자동완성/인라인 타입 힌트가 보이면 정상입니다.

정상 신호

Cargo.toml 자동 인식

프로젝트 폴더를 열면 자동으로 인덱싱 시작 (하단 상태 바에 진행률 표시).

이상 신호

"failed to run rustc" 오류

대부분 링커 미설치가 원인. 07번 슬라이드의 OS별 링커 설치를 다시 확인.

실습

VSCode에서 첫 Rust 프로그램 디버깅하기

01
cargo new hello_rust 후 code hello_rust cargo가 프로젝트 구조와 Cargo.toml을 자동 생성.
02
src/main.rs에 println! 코드 확인 cargo new가 기본 Hello World를 이미 작성해둠.
03
통합 터미널에서 cargo run 컴파일 후 실행까지 한 번에 진행.
04
main 함수에 중단점 찍고 F5 rust-analyzer가 제안하는 "Debug" 코드 렌즈를 눌러 CodeLLDB로 디버깅 연동 확인.

마무리

1주차 들어가기 전 셀프 체크

rustc --version, cargo --version이 오류 없이 출력되나요?

둘 다 rustup이 관리하는 툴체인 버전과 일치해야 해요.

cargo new + cargo run까지 실제로 성공했나요?

이 단계에서 실패하면 rustup이 아니라 링커(OS별 빌드 도구) 문제예요.

내 OS에서 링커로 무엇이 필요한지 설명할 수 있나요?

Windows는 MSVC Build Tools, Linux는 gcc, macOS는 Xcode CLT예요.

VSCode에서 rust-analyzer 인덱싱이 정상 완료됐나요?

하단 상태 바 진행률이 끝나야 자동완성이 정상 동작해요.
