cargo

cargo, Rust의 빌드 도구

Python의 pip + venv, Java의 Maven/Gradle을 합쳐놓은 도구라고 생각하면 편해요. 프로젝트 생성부터 빌드, 실행, 테스트까지 전부 cargo 명령 하나로 처리합니다.

01
cargo new my_project 새 프로젝트를 생성한다. src/main.rs, Cargo.toml이 자동으로 만들어진다.
02
cargo build 소스를 컴파일한다. 결과물은 target/debug/ 아래에 생성된다.
03
cargo run 필요하면 빌드까지 한 뒤 바로 실행한다. 개발 중엔 이 명령을 가장 많이 쓴다.
04
cargo test #[test] 로 표시한 함수들을 모두 찾아 실행한다.

cargo

Cargo.toml 구조

프로젝트의 이름, 버전, 의존성을 관리하는 설정 파일이에요. Python의 pyproject.toml / requirements.txt 역할을 한다고 보면 됩니다.

Cargo.toml

# 패키지 메타 정보 [package] name = "calculator" version = "0.1.0" edition = "2021" # 외부 라이브러리(crate) 의존성 목록 [dependencies]

변수

let, mut, const, 섀도잉

Python 변수는 언제든 자유롭게 재대입할 수 있지만, Rust 변수는 기본적으로 불변(immutable)이에요. 이 부분이 가장 먼저 낯설게 느껴질 거예요.

let

기본 불변 변수

let x = 5; 로 선언하면 x는 이후 값을 바꿀 수 없다. 재대입하면 컴파일 에러.

mut

가변 변수

let mut x = 5; 처럼 mut을 붙여야만 값을 다시 대입할 수 있다.

const

상수

const MAX: u32 = 100; 항상 불변이며, 타입 명시가 필수. 컴파일 타임에 값이 결정돼야 한다.

shadowing

섀도잉

같은 이름으로 let을 다시 써서 새 변수를 만드는 것. mut과 달리 타입도 바꿀 수 있다.

코드 예제

변수 선언 한눈에 보기

src/main.rs

fn main() { // 기본은 불변(immutable) let x = 5; // x = 6; // 컴파일 에러! cannot assign twice // mut을 붙이면 가변 let mut y = 5; y = 6; // OK // 상수는 타입 명시가 필수, 항상 불변 const MAX_POINTS: u32 = 100_000; // 섀도잉: 같은 이름 z를 재선언 let z = 5; let z = z + 1; let z = z * 2; println!("z = {}", z); // 12 }

타입

Rust는 정적 타입 언어

Python은 타입을 신경 쓰지 않아도 실행되지만, Rust는 컴파일 시점에 모든 값의 타입이 확정되어 있어야 해요. 대신 대부분은 타입 추론이 알아서 해줍니다.

정수 / 실수

i32, u32, i64, f64 ...

i는 부호 있는 정수, u는 부호 없는 정수. 뒤 숫자는 비트 수. 정수 기본값은 i32, 실수 기본값은 f64.

bool / char

true/false, 문자 하나

let ok: bool = true; let c: char = '가'; char는 유니코드 스칼라 값 하나(작은따옴표 사용).

튜플

let t: (i32, f64) = (1, 2.0);

서로 다른 타입을 하나로 묶는다. t.0, t.1 처럼 인덱스로 접근한다.

배열

let a: [i32; 3] = [1, 2, 3];

길이가 고정된 같은 타입 값의 모음. Python 리스트와 달리 길이를 늘릴 수 없다.

함수

마지막 expression이 반환값

Python/Java는 return이 있어야 값을 돌려주지만, Rust는 세미콜론 없는 마지막 줄이 그대로 함수의 반환값이 돼요. return은 중간에 일찍 빠져나갈 때만 씁니다.

src/main.rs

fn add(a: i32, b: i32) -> i32 { a + b // 세미콜론이 없다 → 이 expression이 반환값 } fn main() { let sum = add(3, 4); println!("sum = {}", sum); // 7 }

제어문

if도 expression이다

Rust에서 if는 값을 만들어내는 expression이라, 삼항 연산자 없이도 let x = if 조건 { a } else { b }; 형태로 바로 대입할 수 있어요.

if / else

expression으로 사용 가능

let x = if cond { 1 } else { 2 }; 두 분기의 타입은 반드시 같아야 한다.

loop

무한 반복 + break로 값 반환

break 값; 으로 loop 자체가 값을 돌려주는 expression이 될 수 있다.

while

조건이 참인 동안 반복

Python의 while과 동일한 감각. 조건에 괄호는 필요 없다.

for

range / iterator 순회

for i in 0..5 처럼 range를 순회하거나, for x in vec.iter() 로 컬렉션을 순회한다.

코드 예제

제어문 한눈에 보기

src/main.rs

fn main() { let number = 6; // if는 expression → 바로 변수에 바인딩 let desc = if number % 2 == 0 { "짝수" } else { "홀수" }; println!("{}", desc); // loop + break 값으로 결과 반환 let mut counter = 0; let result = loop { counter += 1; if counter == 10 { break counter * 2; } }; println!("result = {}", result); // 20 // for + range, for + iterator for i in 0..5 { print!("{} ", i); // 0 1 2 3 4 } let arr = [10, 20, 30]; for elem in arr.iter() { println!("value: {}", elem); } }

비교

Python 감각으로 보면 낯선 것들

Python

  • 변수는 기본적으로 재대입 자유
  • 실행 중에 타입이 결정 (동적 타입)
  • 들여쓰기로 블록 구분, 세미콜론 없음
  • 인터프리터가 즉시 실행, 별도 빌드 없음

Rust

  • 변수는 기본 불변, mut이 있어야 재대입
  • 컴파일 시점에 타입이 확정 (정적 타입)
  • { } 로 블록 구분, 문장 끝 세미콜론
  • cargo build로 먼저 컴파일해야 실행 가능

흔한 실수

처음엔 다 이 에러를 만난다

01
cannot assign twice to immutable variable let로 선언한 변수에 값을 다시 대입했다. mut을 붙이거나 섀도잉을 사용하자.
02
mismatched types, expected i32, found () 함수의 마지막 반환 expression 뒤에 실수로 세미콜론을 붙였다. 세미콜론이 붙으면 값이 아니라 ()가 된다.
03
expected i32, found f64 정수와 실수를 그냥 더하면 타입이 다르다고 거부당한다. as f64 처럼 명시적으로 캐스팅해야 한다.
04
value borrowed here after move 소유권이 다른 변수로 넘어간 값을 다시 사용했다. 소유권과 빌림은 2~3주차에서 본격적으로 다룬다.

실습

간단한 CLI 계산기 만들기

사용자에게 숫자 두 개와 연산자를 입력받아 사칙연산 결과를 출력하는 프로그램을 만들어보세요.

01
cargo new calculator 새 프로젝트를 생성한다.
02
std::io::stdin()으로 입력 받기 read_line(&mut input)으로 한 줄씩 입력을 받는다.
03
input.trim().parse::<f64>() 문자열을 숫자로 변환한다. 실패할 수 있으니 expect나 match로 처리한다.
04
match로 연산자 분기 +, -, *, / 를 match로 분기하고, 0으로 나누는 경우도 예외로 처리한다.

힌트 · 입력 받기

use std::io; fn main() { let mut input = String::new(); io::stdin() .read_line(&mut input) .expect("입력을 읽지 못했습니다"); let num: f64 = input.trim().parse().expect("숫자가 아닙니다"); println!("입력값: {}", num); }

마무리

이번 주 배운 것, 셀프 체크

아래 질문에 스스로 답해보면서 1주차 내용을 점검해보세요.

cargo new / build / run / test, 각각 언제 쓰는지 설명할 수 있나요?

new는 생성, build는 컴파일, run은 빌드+실행, test는 테스트 실행.

let과 let mut의 차이, 섀도잉이 재대입과 다른 이유를 설명할 수 있나요?

섀도잉은 같은 이름의 새 변수를 만드는 것이라 타입도 바꿀 수 있어요.

함수의 마지막 줄에서 세미콜론을 빼야 하는 이유를 아나요?

세미콜론이 있으면 expression이 아닌 statement가 되어 ()를 반환해요.

if를 expression으로 사용하는 예시를 직접 만들 수 있나요?

let x = if cond { a } else { b }; 두 분기의 타입은 같아야 해요.
