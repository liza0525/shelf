Java 시작 전 환경 설정

VSCode 기준, Windows / Linux / macOS 차이를 항목별로 정리합니다. ← → 키로 페이지를 넘길 수 있어요.

들어가며

JDK 버전과 배포판, 처음에 헷갈리는 지점

Java는 JVM 위에서 돌기 때문에 OS 차이가 C만큼 크진 않지만, "어떤 JDK 배포판을 어디에 설치했고 JAVA_HOME이 어디를 가리키는지"가 OS마다 설정 방식이 달라서 초반에 자주 막힙니다.

헷갈리는 것

  • Oracle JDK vs OpenJDK vs Temurin
  • JAVA_HOME을 어디서 설정하는지 (GUI vs 셸 파일)
  • 여러 버전을 동시에 관리하는 방법

이 자료의 기준

  • Eclipse Temurin(무료, 크로스 플랫폼) LTS 버전 사용
  • Maven 빌드, VSCode Java Extension Pack 기준
  • OS별 JAVA_HOME 설정법을 각각 안내

준비물

설치할 4가지

01
VSCode 에디터 겸 빌드·디버거·테스트 통합 환경.
02
Extension Pack for Java 언어 서버, 디버거, Maven/Gradle, 테스트 러너를 한 번에.

JAVA_HOME 환경 변수 OS별로 설정 위치와 방법이 다름.

VSCode 설치

OS별 설치 방법

Windows

  • 공식 installer(.exe) 실행
  • "PATH에 추가" 체크박스 체크
winget install Microsoft.VisualStudioCode

Linux

  • .deb/.rpm 또는 apt 저장소 등록
  • code 명령어 자동 등록됨
sudo apt install code # 또는 snap install code --classic

macOS

  • .dmg를 Applications로 드래그
  • code 명령은 팔레트에서 별도 등록
brew install --cask visual-studio-code # 이후 Cmd+Shift+P # "Shell Command: Install 'code'"

JDK 설치

Eclipse Temurin 21 (LTS)

Oracle JDK는 상용 라이선스 이슈가 있어, 무료 배포판인 Temurin을 기준으로 안내합니다.

Windows

  • winget 또는 .msi installer
  • installer가 JAVA_HOME도 자동 등록해줌
winget install EclipseAdoptium.Temurin.21.JDK

Linux

  • apt 저장소 이름이 배포판마다 다름
  • SDKMAN으로도 설치 가능
sudo apt install openjdk-21-jdk # Fedora: sudo dnf install java-21-openjdk-devel

macOS

  • brew cask로 설치
  • /Library/Java/JavaVirtualMachines에 등록됨
brew install --cask temurin@21

환경 변수

JAVA_HOME, OS마다 설정 방법이 다르다

Windows

  • 시스템 속성 → 환경 변수 GUI 사용 가능
  • 또는 관리자 PowerShell에서 setx
setx JAVA_HOME "C:\Program Files\Eclipse Adoptium\jdk-21" /M

Linux

  • ~/.bashrc 또는 ~/.profile에 export
  • update-alternatives로 버전 전환 가능
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk export PATH=$JAVA_HOME/bin:$PATH

macOS

  • ~/.zshrc에 추가
  • java_home 유틸로 버전별 경로 조회
export JAVA_HOME=$(/usr/libexec/java_home -v 21) export PATH=$JAVA_HOME/bin:$PATH

빌드 도구

Maven 설치, 여러 버전 관리

Gradle은 프로젝트에 gradlew 래퍼가 포함되면 별도 설치가 필요 없지만, Maven은 보통 시스템에 직접 설치합니다.

Windows

  • winget으로 설치
  • SDKMAN 미지원 (WSL/Git Bash 안에서만 가능)
winget install Apache.Maven

Linux

  • apt 또는 SDKMAN
  • SDKMAN으로 여러 버전 전환 편함
sudo apt install maven # 또는 curl -s "https://get.sdkman.io" | bash

macOS

  • brew 또는 SDKMAN
  • SDKMAN 스크립트는 Linux와 동일
brew install maven

설치 확인

버전 체크로 마무리

터미널 (모든 OS 공통 명령)

java -version javac -version mvn -version # mvn -version 출력의 "Java version" 줄이 # 방금 설치한 JDK 21을 가리키는지 확인

java -version이 다른 버전을 가리키면 JAVA_HOME/PATH 우선순위 문제일 확률이 높습니다.

자주 걸리는 문제

OS별 트러블슈팅

Win
java -version은 새 버전, mvn -version은 옛 버전 시스템 PATH에 여러 JDK 경로가 남아있는 경우. 환경 변수 편집 창에서 오래된 JDK 경로를 지운다.
Linux
배포판 기본 저장소의 JDK가 최신이 아님 Ubuntu LTS 등은 openjdk 패키지 버전이 고정돼 있을 수 있음. Temurin PPA나 SDKMAN으로 최신 LTS를 받는다.
macOS
brew로 설치했는데 java_home이 못 찾음 brew openjdk는 /opt/homebrew에만 설치되고 /Library/Java에는 심볼릭 링크가 없음. brew info의 caveats에 안내된 ln -s 명령을 실행해야 함.
공통
여러 JDK 버전이 충돌 Linux/macOS는 SDKMAN(sdk use java 21-tem)으로, Windows는 환경 변수를 직접 바꿔가며 전환.

실행 설정

VSCode에서 실행 환경 확인하기

여러 JDK가 설치돼 있으면 VSCode가 엉뚱한 버전을 쓸 수 있어요. settings.json에서 명시적으로 지정합니다.

.vscode/settings.json

{ "java.configuration.runtimes": [ { "name": "JavaSE-21", // Windows 경로 예시 "path": "C:\\Program Files\\Eclipse Adoptium\\jdk-21", "default": true } ] }

실습

VSCode에서 첫 Java 프로그램 실행하기

01
빈 폴더 생성 후 code . 로 VSCode 열기 mkdir hello-java && cd hello-java && code .
02
Command Palette → "Java: Create Java Project" Maven 기반 프로젝트 템플릿을 자동 생성.
03
App.java의 main에서 Run 클릭 에디터 상단의 "Run"이 뜨면 코드 렌즈가 정상 동작 중.
04
F5로 중단점 디버깅해보기 변수 값이 디버그 패널에 보이면 연동 성공.

마무리

1주차 들어가기 전 셀프 체크

java -version과 javac -version이 같은 버전을 가리키나요?

버전이 다르면 JDK와 JRE가 따로 설치돼 있을 수 있어요.

내 OS에서 JAVA_HOME을 어디서 바꾸는지 알고 있나요?

Windows는 시스템 속성 GUI/setx, Linux·macOS는 셸 설정 파일.

VSCode에서 Extension Pack for Java가 정상 설치됐나요?

프로젝트 사이드바에 Maven/Java Projects 뷰가 보여야 해요.

mvn -version이 방금 설치한 JDK를 가리키나요?

다른 버전을 가리키면 PATH 우선순위 문제입니다.
