Arrays & Pointers · Decay

배열은 함수에 들어가는 순간 포인터가 된다

배열 이름은 대부분의 식에서 첫 원소를 가리키는 포인터로 decay된다. 함수 인자로 넘기면 배열 전체가 아니라 포인터 한 개만 복사되어 전달된다.

sizeof가 다르게 나오는 이유

void check_size(int arr[]) { // arr는 사실 int* 파라미터일 뿐이다 printf("%zu\n", sizeof(arr)); // 8 (포인터 크기) } int main(void) { int nums[10]; printf("%zu\n", sizeof(nums)); // 40 (10 * sizeof(int)) check_size(nums); // nums가 &nums[0]으로 decay }

Pointer Arithmetic

포인터 연산은 바이트가 아니라 원소 단위다

ptr + n은 ptr의 주소에 n * sizeof(*ptr) 바이트를 더한다. 같은 배열을 가리키는 두 포인터를 빼면 그 사이의 원소 개수가 나온다.

포인터 연산 예제

int nums[5] = {10, 20, 30, 40, 50}; int *p = nums; printf("%d\n", *(p + 2)); // 30, nums[2]와 동일 int *q = &nums[4]; ptrdiff_t diff = q - p; // 4 (원소 개수 차이, 바이트 아님) printf("%td\n", diff); p++; // p는 8바이트가 아니라 sizeof(int)만큼 이동 printf("%d\n", *p); // 20

const 포인터 조합

무엇이 const인지 정확히 구분하자

const의 위치에 따라 "가리키는 값"이 고정인지 "포인터 자체"가 고정인지가 달라진다. 오른쪽에서 왼쪽으로 읽으면 헷갈리지 않는다.

const int *p

가리키는 값이 상수

p가 다른 주소를 가리키도록 바꿀 수는 있지만, *p를 통해 값을 수정할 수는 없다. "포인터를 통해 읽기 전용"인 경우.

int * const p

포인터 자체가 상수

*p로 값을 바꾸는 건 가능하지만, p가 다른 주소를 가리키도록 재대입할 수는 없다. 선언과 동시에 초기화해야 한다.

const int * const p

둘 다 상수

값도 못 바꾸고 포인터도 못 바꾼다. 읽기 전용 참조를 넘기고 싶을 때 함수 시그니처에서 자주 쓰인다.

다중 포인터 · int **

포인터를 통해 포인터의 값을 바꾸기

함수 안에서 "포인터 변수 자체"를 바꾸고 싶다면 포인터의 포인터, 즉 int **가 필요하다. 단순히 int *를 넘기면 그 함수는 사본만 받는다.

malloc 결과를 함수 밖으로 전달하는 패턴

void allocate(int **out, int n) { *out = malloc(n * sizeof(int)); // out이 가리키는 포인터 변수를 교체 if (*out) { for (int i = 0; i < n; i++) (*out)[i] = i; } } int main(void) { int *arr = NULL; allocate(&arr, 5); // arr의 주소를 넘겨야 arr 자체를 바꿀 수 있다 printf("%d\n", arr[3]); // 3 free(arr); }

함수 포인터 · void (*fp)(int)

함수도 값처럼 가리킬 수 있다

함수 포인터는 함수의 주소를 담는 변수다. 괄호 위치가 중요한데, *fp를 괄호로 묶지 않으면 "함수를 반환하는 함수"로 잘못 해석된다.

선언, 대입, 호출

void greet(int times) { for (int i = 0; i < times; i++) printf("hi\n"); } int main(void) { void (*fp)(int) = greet; // fp는 int를 받고 void를 반환하는 함수를 가리킴 fp(3); // greet(3)과 동일하게 동작 (*fp)(3); // 명시적으로 역참조해도 결과는 같다 }

void* · 제네릭 스타일

void*는 C식 제네릭이다

void*는 어떤 타입이든 가리킬 수 있지만 역참조 전에는 타입 정보가 없다. 표준 라이브러리의 qsort는 이 방식으로 어떤 타입의 배열이든 정렬한다.

qsort 비교 함수 구현

int cmp_int(const void *a, const void *b) { int ia = *(const int *)a; // void*를 원래 타입으로 캐스팅 후 역참조 int ib = *(const int *)b; return (ia > ib) - (ia < ib); // 양수/음수/0 반환 } int main(void) { int nums[] = {5, 2, 9, 1, 3}; qsort(nums, 5, sizeof(int), cmp_int); // qsort는 원소 크기를 몰라도 void*와 size_t만으로 동작 }

Python과의 차이

Python에는 애초에 이런 고민이 없다

Python의 모든 변수는 이미 객체에 대한 참조이고, 함수도 일급 객체라 그냥 변수에 담아 넘기면 된다. 리스트는 타입에 상관없이 아무 값이나 담을 수 있다. C는 이 편리함을 언어가 대신 해주지 않는 대신, 포인터라는 명시적인 도구로 같은 일을 하게 만든다.

const int *, int **, void (*)(int), void* 같은 문법이 낯설게 느껴지는 건 당연하다. 이건 Python에는 대응 개념이 없는, C 고유의 "메모리를 직접 다루는 어휘"이기 때문이다. 이 어휘에 익숙해지는 것이 이번 주차의 핵심이다.

실습 미리보기 · 커맨드 디스패치

함수 포인터 배열로 커맨드를 분기하기

if/else나 switch로 명령어를 분기하는 대신, 이름과 함수 포인터를 짝지은 테이블을 만들면 명령어 추가가 테이블에 한 줄 더하는 일이 된다.

디스패치 테이블 골격

typedef void (*cmd_fn)(void); void cmd_hello(void) { printf("hello!\n"); } void cmd_bye(void) { printf("bye!\n"); } typedef struct { const char *name; cmd_fn fn; } command_t; command_t table[] = { {"hello", cmd_hello}, {"bye", cmd_bye}, }; // 실습: 문자열을 입력받아 table을 순회하며 이름이 일치하는 fn을 호출해보기

흔한 실수 · Pitfalls

포인터 코드에서 자주 만나는 함정

1
선언 우선순위를 헷갈림 int *arr[10]은 "int 배열을 가리키는 포인터"가 아니라 "포인터 10개짜리 배열"이다. []가 *보다 우선순위가 높기 때문. 헷갈리면 괄호로 명시하자: int (*arr)[10].
2
decay 이후 sizeof를 잘못 계산 함수 파라미터로 받은 배열에 sizeof(arr) / sizeof(arr[0])를 쓰면 배열 전체 크기가 아니라 항상 1이 나온다. arr은 이미 포인터라서 sizeof(arr)이 포인터 크기이기 때문. 크기는 별도 인자로 넘겨야 한다.
3
dangling pointer를 반환 지역 배열의 주소를 return하면 함수가 끝나며 스택 프레임이 해제된 이후의 주소를 돌려주는 셈이다. 힙에 malloc하거나 static으로 선언해야 함수 밖에서도 유효하다.

요약 · Summary

이번 주차에서 챙겨야 할 것

배열은 함수 인자로 넘어가면 포인터로 decay된다
포인터 연산은 원소 단위, 바이트 단위가 아니다
const 위치가 "값" const인지 "포인터" const인지를 결정한다
int **는 포인터 변수 자체를 함수 밖에서 바꿀 때 필요하다
함수 포인터와 void*를 결합하면 qsort류의 제네릭 코드를 만들 수 있다
